# 기름값 헌터 실행용 OKR

기준일: 2026-03-08

적용 기간: 2026 Q2 ~ 2027 Q1

문서 목적:

- 제품 우선순위를 `유입`, `반복 사용`, `수익화`, `B2B 검증` 순서로 정렬한다.
- 분기별 목표와 측정 기준을 명확히 해 실행 속도를 높인다.
- 기능 추가가 아니라 사업 성과 기준으로 의사결정을 내리기 위한 운영 문서로 사용한다.

## 1. 연간 전략 요약

전략 문장:

`SEO 유입형 무료 도구 -> 반복 사용 서비스 -> B2B 운영 도구`

핵심 가설:

- 개인 사용자는 검색 도구로 유입되고, 반복 사용을 만드는 기능이 붙어야 트래픽 자산이 된다.
- 개인 직접 과금보다 광고, 제휴, B2B가 더 현실적인 수익원이다.
- 장기 경쟁력은 단순 검색 UI가 아니라 `생활권별 주유 가격 패턴 데이터`와 `운전자 행동 데이터`에서 나온다.

## 2. 연간 Company Objectives

### Objective A. 무료 유입을 반복 사용 서비스로 전환한다

성공 상태:

- 검색 후 이탈하는 도구가 아니라 주간 재방문이 발생하는 서비스가 된다.
- 사용자가 위치, 주유 패턴, 관심 주유소를 저장하기 시작한다.

연간 Key Results:

- 월간 활성 사용자 중 주간 재방문 사용자 비율 `25%` 달성
- 검색 실행 사용자 대비 지도 클릭 전환율 `30%` 달성
- 저장 기능 사용 사용자 비율 `20%` 달성
- 알림 opt-in 사용자 `5,000`명 확보

### Objective B. 광고 외 매출원을 만든다

성공 상태:

- AdSense만 붙은 사이트가 아니라 제휴와 파일럿 매출이 함께 발생한다.
- 매출 실험이 제품 구조 안에 자연스럽게 들어간다.

연간 Key Results:

- 광고 외 월 매출 발생
- 제휴 실험 3건 이상 진행
- 광고 외 매출 비중 `30%` 달성

### Objective C. B2B 문제를 검증하고 첫 유료 고객을 만든다

성공 상태:

- 어떤 조직이 왜 이 서비스를 돈 내고 쓰는지가 정의된다.
- 인터뷰에서 끝나지 않고 유료 파일럿까지 연결된다.

연간 Key Results:

- B2B 인터뷰 `20`건 완료
- 파일럿 제안서 `10`건 발송
- 유료 파일럿 고객 `3`곳 확보

## 3. 운영 North Star와 Guardrails

North Star:

- `주간 재방문 검색 사용자 수`

Primary KPIs:

- 검색 실행 수
- 검색 실행 대비 결과 노출 비율
- 검색 실행 대비 지도 클릭 비율
- 저장 기능 사용률
- 알림 등록률
- 7일 재방문율
- 30일 재방문율
- 광고 매출
- 제휴 매출
- 유료 파일럿 수

Guardrails:

- 검색 결과 화면 신뢰도를 훼손하는 광고 배치는 하지 않는다.
- 앱 개발은 웹 리텐션 검증 이전에 시작하지 않는다.
- 소비자 구독을 주 수익모델로 잡지 않는다.
- 데이터 자산이 충분히 쌓이기 전 API 사업을 열지 않는다.

## 6. 측정 정의

`검색 실행 수`

- 사용자가 검색 버튼을 눌러 `/api/gas` 호출을 시작한 횟수

`지도 클릭 전환율`

- `map_click / search_executed`

`저장 기능 사용률`

- 저장 기능을 1회 이상 사용한 사용자 수 / 검색 사용자 수

`7일 재방문율`

- 첫 사용 후 7일 내 다시 검색을 실행한 사용자 비율

`알림 등록률`

- 알림 opt-in 사용자 수 / 저장 기능 사용자 수

`광고 외 매출 비중`

- 제휴 + 파일럿 + 기타 직접 매출 / 전체 매출
